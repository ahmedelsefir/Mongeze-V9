
# 📘 المنجز V9: المحرك الرسمي المحسن (The Strategic Engine)
### نظام الإدارة اللوجستية والربط السحابي الذكي 🚀

---

## 🌍 نظرة عامة (Overview)
هذا المشروع هو **العمود الفقري الرقمي** لمنظومة "المنجز". يمثل النظام حلقة الوصل الذكية بين أربعة أركان أساسية في السوق: (**العميل، الموظف، المحاسب، والمندوب**) [cite: 2026-03-03]. 

**كيف يعمل النظام؟ (؟)** يعمل بكفاءة سحابية كاملة تربط بين معالجة اللغة الطبيعية (**Gemini AI**) لذكاء الردود، والبيانات اللحظية (**Cloud Firestore**) لسرعة التنفيذ.

---

## 🛠️ المكونات الاستراتيجية (System Core Components)

### 1. بوابة الإدارة المركزية (Admin/Accountant Portal) 📊
* **الرقابة المباشرة:** تمكن المحاسبين من مراقبة التدفق المالي اللحظي.
* **المحرك المالي (؟):** تطبيق آلي لضريبة القيمة المضافة (**VAT 14%**) وحساب العمولات بناءً على بيانات المستخدمين المسجلة في السحابة.

### 2. رادار المندوب الذكي (Driver's GPS Radar) 🛰️
* **التتبع الجغرافي (؟):** يعتمد على مكتبة `streamlit-js-eval` لتعقب الموقع الجغرافي للمندوب لحظة بلحظة لضمان الدقة.
* **ذكاء التوزيع:** عرض الطلبات النشطة (مثل طلب "توصيل أكل") بناءً على الحالة السحابية `pending` المستخرجة من Firestore.

### 3. مساعد الذكاء الاصطناعي (AI Assistant) 🧠
* **العقل المفكر (؟):** دمج نموذج `gemini-1.5-flash` للرد على استفسارات الموظفين والعملاء وتوجيههم داخل النظام آلياً.

### 4. الأمن والخصوصية (Production-Grade Security) 🔑
* **صندوق الأسرار (؟):** إدارة المفاتيح الحساسة عبر نظام `Streamlit Secrets` لضمان حماية الـ API Keys وبيانات Firebase من أي اختراق.

## 📊 هيكلية البيانات (Database Schema)
**كيف يتم تنظيم المعلومات في السحابة؟ (؟)**

* **Collection (users):** تخزين الرتب (`admin`, `driver`, `customer`) والبيانات التعريفية لكل قائد أو عميل.
* **Collection (orders):** إدارة حالة الطلبات من "قيد الانتظار" إلى "تم التوصيل" مع طابع زمني دقيق لكل حركة.
